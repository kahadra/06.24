Prior Approaches

초기 배경 저장

처음의 빈 장면을 기준 배경으로 둔다.

시간이 지나면 장면 변화에 취약하다.

시간 평균

최근 프레임을 평균내며 천천히 적응한다.

느린 물체와 반복 배경에 약하다.

단일 Gaussian

픽셀마다 평균과 분산을 둔다.

한 픽셀의 여러 정상 상태를 설명하지 못한다.

기존 방법은 각각 단순하고 빠르지만, 한 픽셀이 여러 정상 상태를 갖는 상황을 설명하지 못합니다.
Core Idea

한 픽셀이 여러 정상값을 기억한다

R value
G value

Gaussian 하나

이 픽셀에서 자주 보였던 색의 덩어리

Mixture

그런 색 덩어리 여러 개의 목록

자주 보이고 안정적인 덩어리는 배경 후보가 된다.

픽셀 하나를 작은 관찰자로 보면 됩니다. 충분히 오래 보면 값들이 아무렇게나 흩어지는 것이 아니라 몇 개의 덩어리로 모입니다.
Model

픽셀값의 확률을 Gaussian mixture로 본다

P(Xt) = sum_i^K w_i,t * eta(Xt, mu_i,t, Sigma_i,t) Sigma_k,t = sigma_k,t^2 * I

모든 픽셀이 자기만의 작은 mixture model을 가진다.

K

기억할 후보 수

w

얼마나 자주 보였나

mu

색 덩어리의 중심

sigma

값의 흔들림

계산량 절감

full covariance 대신 sigma squared I로 단순화한다.

현재 픽셀값 Xt가 관측될 확률을 K개의 Gaussian이 섞인 형태로 둡니다. 실시간 처리가 중요하기 때문에 covariance를 단순화합니다.
Online Update

판단과 학습이 동시에 일어난다

Input
현재 픽셀값 Xt
Match?
기존 평균에서 2.5 sigma 이내
Update
match된 평균과 분산 갱신
Replace
없으면 가장 약한 component 교체
Normalize
weight를 다시 정규화
w_k,t = (1 - alpha) * w_k,t-1 + alpha * M_k,t rho = alpha * eta(Xt | mu_k, sigma_k)

현재 프레임을 분류하면서, 그 결과로 다음 프레임의 배경 기억을 수정한다.

매 프레임에서 픽셀값이 들어오면 기존 Gaussian 중 하나와 충분히 가까운지 봅니다. match가 있으면 갱신하고, 없으면 가장 약한 component를 교체합니다.
Background Selection

자주 보이고 안정적인 component부터

Component Aw / sigma 높음
Component B반복 배경
Component C전경 후보
누적 weightthreshold T
score_k = w_k / sigma_k B = argmin_b (sum_k=1^b w_k > T)

판단 규칙

현재 픽셀값이 선택된 background component에 match되면 배경, 아니면 전경이다.

component를 w 나누기 sigma 기준으로 정렬합니다. 자주 보였고 안정적인 component일수록 배경 가능성이 높습니다.

Parameters

기억 속도와 배경의 폭

K: 기억 용량

적은 모드많은 모드

한 픽셀이 기억할 수 있는 후보 수.

alpha: 적응 속도

오래 기억빨리 적응

작으면 안정적이고, 크면 전경 흡수 위험이 커진다.

T: 배경 다양성

보수적넓게 인정

반복 배경을 받아들이지만 너무 높으면 전경도 배경화될 수 있다.

2.5 sigma: local threshold

좁은 수용넓은 수용

전역 threshold가 아니라 component별 분산에 따라 달라진다.

K는 기억할 상태 수, alpha는 기억의 속도, T는 배경의 폭, 2.5 sigma는 Gaussian별 local threshold입니다.
Why It Works

세 가지 현실 상황을 한 구조에서 처리한다

반복 배경

물결, 나뭇잎, 반사처럼 계속 바뀌지만 반복되는 상태를 여러 component로 보관한다.

배경은 하나일 필요가 없다.

느린 조명 변화

online update로 평균과 weight가 서서히 이동한다.

시간 변화에 모델이 따라간다.

배경 복구

원래 배경 component가 낮은 weight로 남아 있으면 객체가 사라진 뒤 회복된다.

과거 배경을 완전히 버리지 않는다.

이 모델은 반복 배경, 느린 조명 변화, 사라진 배경의 복구를 한 구조 안에서 설명합니다.

Limitations

픽셀 통계만으로는 부족한 것들

픽셀 독립

주변 구조와 형태를 직접 보지 않는다.

그림자

명시적 shadow removal이 아니다.

겹침과 교차

connected component와 tracking이 혼동된다.

급격한 변화

alpha보다 빠르면 안정화 시간이 필요하다.

의미를 이해하는 모델이 아니라, 픽셀 색 통계로 현실 배경을 견디는 모델이다.

이 모델은 강력한 고전이지만 의미 이해, 그림자 분리, 객체 겹침, 급격한 조명 변화에는 한계가 있습니다.
